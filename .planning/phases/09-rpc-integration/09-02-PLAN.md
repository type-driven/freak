**rpc_test.ts -- SC-1: RPC call returns typed response (automated, in-process)**

Use `RpcTest.makeClient` for in-process testing (no HTTP needed, per research recommendation). This is cleaner than FakeServer for RPC.

1. Define a test RpcGroup:
   ```typescript
   import { Rpc, RpcGroup, RpcTest } from "effect/unstable/rpc";
   import { Effect, Layer, Schema } from "effect";

   const ItemSchema = Schema.Struct({ id: Schema.String, name: Schema.String });

   const ListItems = Rpc.make("ListItems", {
     success: Schema.Array(ItemSchema),
   });

   const CreateItem = Rpc.make("CreateItem", {
     payload: { name: Schema.String },
     success: ItemSchema,
   });

   const TestRpc = RpcGroup.make(ListItems, CreateItem);
   ```

2. Define handler layer:
   ```typescript
   const TestHandlers = TestRpc.toLayer({
     ListItems: () => Effect.succeed([{ id: "1", name: "Widget" }]),
     CreateItem: ({ name }) => Effect.succeed({ id: "2", name }),
   });
   ```

3. Write SC-1 test using `RpcTest.makeClient`:
   ```typescript
   Deno.test("SC-1: RPC call returns typed response via RpcTest.makeClient", async () => {
     await Effect.scoped(
       Effect.gen(function*() {
         const client = yield* RpcTest.makeClient(TestRpc);
         const items = yield* client.ListItems({});
         assertEquals(items, [{ id: "1", name: "Widget" }]);

         const created = yield* client.CreateItem({ name: "Gadget" });
         assertEquals(created, { id: "2", name: "Gadget" });
       })
     ).pipe(
       Effect.provide(TestHandlers),
       Effect.runPromise,
     );
   });
   ```

4. Write dispose test:
   ```typescript
   Deno.test("rpc dispose: app.dispose() succeeds without error", async () => {
     const app = createEffectApp({ layer: Layer.empty });
     app.rpc({
       group: TestRpc,
       path: "/rpc/test",
       protocol: "http",
       handlerLayer: TestHandlers,
     });
     app.handler();
     await app.dispose();
   });
   ```

5. Write WS registration smoke test (catches broken WS mounts programmatically -- SC-2 basic validation):

     const app = createEffectApp({ layer: Layer.empty });

**rpc_types_test.ts -- SC-3: TypeScript rejects undeclared procedure**

Model this on the existing `types_test.ts` pattern. The key is that `RpcClient.make(group)` returns a typed client where only declared procedure tags exist as properties.

```typescript
import { Rpc, RpcGroup, RpcTest } from "effect/unstable/rpc";
import { Effect, Schema } from "effect";

const ListItems = Rpc.make("ListItems", {
  success: Schema.Array(Schema.String),
});

const TestRpc = RpcGroup.make(ListItems);

const TestHandlers = TestRpc.toLayer({
  ListItems: () => Effect.succeed(["item1"]),
});

Deno.test("SC-3: TypeScript rejects call to undeclared procedure", async () => {
  await Effect.scoped(
    Effect.gen(function*() {
      const client = yield* RpcTest.makeClient(TestRpc);

      // Valid call compiles
      yield* client.ListItems({});

      // @ts-expect-error -- "NonExistent" is not a declared procedure in TestRpc
      client.NonExistent;
    })
  ).pipe(
    Effect.provide(TestHandlers),
    Effect.runPromise,
  );
});
```

Run with `deno check` to verify the `@ts-expect-error` directive fires (meaning TS would reject the invalid access). If `@ts-expect-error` does NOT fire (meaning the property access is valid), the test file itself will have a type error, which is correct behavior for catching regressions.

Alternative approach if RpcClient property access doesn't work with `@ts-expect-error`: use `expectTypeOf` from `expect-type` to assert the client type does not have the property.

**services/rpc.ts -- TodoRpc group definition + handlers:**

Define an RPC group for todo CRUD + a streaming watch:

```typescript
import { Rpc, RpcGroup } from "effect/unstable/rpc";
import { Effect, Schema, Stream, SubscriptionRef } from "effect";
import { TodoSchema } from "../types.ts";
import { TodoService } from "./TodoService.ts";

// Error schema for typed errors
const TodoError = Schema.TaggedStruct("TodoError", { message: Schema.String });

// Procedures
const ListTodos = Rpc.make("ListTodos", {
  success: Schema.Array(TodoSchema),
});

const CreateTodo = Rpc.make("CreateTodo", {
  payload: { text: Schema.String },
  success: TodoSchema,
});

const DeleteTodo = Rpc.make("DeleteTodo", {
  payload: { id: Schema.String },
  success: Schema.Void,
  error: TodoError,
});

// Streaming procedure for live updates
const WatchTodos = Rpc.make("WatchTodos", {
  success: Schema.Array(TodoSchema),
  stream: true,
});

export const TodoRpc = RpcGroup.make(ListTodos, CreateTodo, DeleteTodo, WatchTodos);

// Handler implementations
export const TodoRpcHandlers = TodoRpc.toLayer({
  ListTodos: () =>
    Effect.gen(function*() {
      const svc = yield* TodoService;
      return yield* svc.list();
    }),
  CreateTodo: ({ text }) =>
    Effect.gen(function*() {
      const svc = yield* TodoService;
      return yield* svc.create(text);
    }),
  DeleteTodo: ({ id }) =>
    Effect.gen(function*() {
      const svc = yield* TodoService;
      return yield* svc.remove(id);
    }),
  WatchTodos: () =>
    // Simple polling stream: emit current todo list every 2 seconds
    // A production app would use a SubscriptionRef or event bus for real push
    Stream.repeatEffectWithSchedule(
      Effect.gen(function*() {
        const svc = yield* TodoService;
        return yield* svc.list();
      }),
      Schedule.spaced("2 seconds"),
    ),
});
```

Note: The WatchTodos stream implementation may need adjustment based on available imports. If `Schedule` is not easily available, a simpler `Stream.fromEffect` approach works. The key is demonstrating the streaming protocol -- the exact push mechanism is secondary.

**main.ts -- Restructure to support app.rpc() void calls:**

The current main.ts uses a single builder chain:
```typescript
export const app = createEffectApp({ layer: AppLayer })
  .httpApi("/api", TodoApi, TodosWithDeps)
  .use(staticFiles())
  .fsRoutes();
```

Since `app.rpc()` returns `void` (not `this`), the chain must be restructured. Capture the EffectApp in a variable, call `httpApi()` and `rpc()` as standalone statements, then continue the chain for Fresh methods that return `this`.

The full restructured main.ts:

```typescript
import { HttpError, staticFiles } from "@fresh/core";
import { createEffectApp } from "@fresh/effect";
import { Cause, Layer } from "effect";
import { AppLayer } from "./services/layers.ts";
import { NotFoundError } from "./services/errors.ts";
import { TodoApi, TodosLive } from "./services/api.ts";
import { TodoRpc, TodoRpcHandlers } from "./services/rpc.ts";

// Pre-compose handler layers with AppLayer so service dependencies are available
const TodosWithDeps = Layer.provide(TodosLive, AppLayer);
const RpcWithDeps = Layer.provide(TodoRpcHandlers, AppLayer);

const app = createEffectApp({
  layer: AppLayer,
  mapError: (cause) => {
    const defect = Cause.squash(cause as never);
    if (defect instanceof NotFoundError) {
      throw new HttpError(404);
    }
    throw new HttpError(500);
  },
});

// Mount HttpApi (returns this, but we don't chain)
app.httpApi("/api", TodoApi, TodosWithDeps);

// Mount RPC -- HTTP protocol for request/response (list, create, delete)
app.rpc({
  group: TodoRpc,
  path: "/rpc/todos",
  protocol: "http",
  handlerLayer: RpcWithDeps,
});

// Mount RPC -- WebSocket protocol for streaming (watch)
app.rpc({
  group: TodoRpc,
  path: "/rpc/todos/ws",
  protocol: "websocket",
  handlerLayer: RpcWithDeps,
});

// Continue with Fresh middleware and routes
app.use(staticFiles()).fsRoutes();

// Re-export for Fresh's dev server
export { app };
```

Key changes from the current main.ts:
1. `const app = createEffectApp(...)` captures result (no longer chained through `.httpApi()`)
2. `app.httpApi(...)` is a standalone statement (still works -- httpApi returns `this` but we ignore it)
3. `app.rpc(...)` calls are standalone (returns void, cannot chain)
4. `app.use(staticFiles()).fsRoutes()` is the final chain (these return `this`)
5. `export { app }` at the end (named re-export, same as current `export const app = ...`)

**islands/RpcDemo.tsx -- Island using useRpcResult:**

```tsx
import { useState } from "preact/hooks";
import { useRpcResult } from "@fresh/effect/island";
import { TodoRpc } from "../services/rpc.ts";

export default function RpcDemo() {
  const [result, client] = useRpcResult(TodoRpc, { url: "/rpc/todos" });
  const [text, setText] = useState("");

  const loadTodos = () => client.ListTodos({});
  const createTodo = () => {
    if (text.trim()) {
      client.CreateTodo({ text: text.trim() });
      setText("");
    }
  };

  return (
    <div>
      <h2>RPC Todo List</h2>
      <div>
        <input
          type="text"
          value={text}
          onInput={(e) => setText((e.target as HTMLInputElement).value)}
          placeholder="New todo..."
        />
        <button onClick={createTodo}>Add</button>
        <button onClick={loadTodos}>Refresh</button>
      </div>

      {result._tag === "idle" && <p>Click Refresh to load todos</p>}
      {result._tag === "loading" && <p>Loading...</p>}
      {result._tag === "ok" && (
        <ul>
          {Array.isArray(result.value)
            ? result.value.map((todo: any) => (
                <li key={todo.id}>{todo.text}</li>
              ))
            : <li>Created: {(result.value as any).text}</li>
          }
        </ul>
      )}
      {result._tag === "err" && <p style="color:red">Error: {String(result.error)}</p>}
    </div>
  );
}
```
