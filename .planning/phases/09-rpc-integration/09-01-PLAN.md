3. Add `rpc()` method on EffectApp. Signature:
   ```typescript
   rpc(options: {
     group: any,
     path: string,
     protocol: "http" | "websocket",
     handlerLayer: any,
   }): void
   ```

   Implementation:

   a. Build the RPC layer with `path: "/"` (relative root, same strategy as httpApi):
      ```typescript
      const serverLayer = RpcServer.layerHttp({
        group: options.group,
        path: "/",  // Routes are relative to the mount prefix
        protocol: options.protocol,
      }).pipe(
        Layer.provide(options.handlerLayer),
        Layer.provide(
          options.protocol === "http"
            ? RpcSerialization.layerJson
            : RpcSerialization.layerNdjson
        ),
        Layer.provide(HttpServer.layerServices),
      );
      ```

   b. Convert to web handler with shared memoMap:
      ```typescript
      const { handler, dispose } = HttpRouter.toWebHandler(serverLayer as any, {
        memoMap: this.#runtime.memoMap,
      });
      ```

   c. Store disposer: `this.#rpcDisposers.push(dispose);`

   d. **Mount routes on Fresh app. Definitive WS + HTTP strategy:**

      For BOTH protocols, mount at `options.path + "/*"` WITH prefix stripping (same as httpApi). This handles POST requests for HTTP and sub-path requests for WS:

      ```typescript
      this.#app.all(options.path + "/*", async (ctx) => {
        const url = new URL(ctx.req.url);
        url.pathname = url.pathname.slice(options.path.length) || "/";
        const rewritten = new Request(url.toString(), ctx.req);
        return await (handler as any)(rewritten);
      });
      ```

      For WebSocket protocol ADDITIONALLY mount at the EXACT path WITHOUT prefix stripping. The WS upgrade handshake arrives as a GET request at the exact path. Since `RpcServer.layerHttp` was given `path: "/"`, we rewrite the pathname to "/" to match the internal registration:

      ```typescript
      if (options.protocol === "websocket") {
        this.#app.all(options.path, async (ctx) => {
          const url = new URL(ctx.req.url);
          url.pathname = "/";
          const rewritten = new Request(url.toString(), ctx.req);
          return await (handler as any)(rewritten);
        });
      }
      ```

      This dual registration is necessary because Fresh's `path + "/*"` glob does NOT match the exact path itself (only paths with a trailing segment), while the WS upgrade hits the exact path as GET. The inner Effect router sees "/" in both cases, which matches the `path: "/"` given to `RpcServer.layerHttp`.

   Use `// deno-lint-ignore no-explicit-any` where needed (same as httpApi).

**Type definitions:**

```typescript
type RpcResultState<A, E> =
  | { readonly _tag: "idle" }
  | { readonly _tag: "loading" }
  | { readonly _tag: "ok"; readonly value: A }
  | { readonly _tag: "err"; readonly error: E };

type RpcStreamState<A, E> =
  | { readonly _tag: "connecting" }
  | { readonly _tag: "connected"; readonly latest: A | null }
  | { readonly _tag: "error"; readonly error: E }
  | { readonly _tag: "closed" };
```

**useRpcResult implementation:**

Returns `[state, client]` where:
- `state` is `RpcResultState` (starts as `{ _tag: "idle" }`)
- `client` is a proxy object whose method names match the RpcGroup's procedure tags

Each method call on `client`:
1. Sets state to `{ _tag: "loading" }`
2. Creates a scoped Effect that:
   - Gets `RpcClient.make(group)`
   - Calls the procedure on the client
   - Maps success to `{ _tag: "ok", value }` and error to `{ _tag: "err", error }`
3. Runs the Effect with the HTTP protocol layer:
   ```typescript
   const layer = Layer.mergeAll(
     RpcClient.layerProtocolHttp({ url: options.url }),
     RpcSerialization.layerJson,
     FetchHttpClient.layer,
   );
   ```
4. Uses `Effect.scoped` + `Effect.provide(layer)` + `Effect.runPromise`

The client proxy is created via `Proxy` that intercepts property access and returns an async function:

```typescript
const client = new Proxy({} as any, {
  get: (_target, prop: string) => (payload: any) => {
    setState({ _tag: "loading" });
    const effect = Effect.scoped(
      Effect.gen(function* () {
        const c = yield* RpcClient.make(group);
        return yield* (c as any)[prop](payload);
      })
    ).pipe(
      Effect.provide(layer),
    );
    Effect.runPromise(effect).then(
      (value) => { if (!unmountedRef.current) setState({ _tag: "ok", value }); },
      (error) => { if (!unmountedRef.current) setState({ _tag: "err", error }); },
    );
  },
});
```

Use `useRef` for unmounted flag, clean up in `useEffect` return.

**useRpcStream implementation:**

On mount:
1. Sets state to `{ _tag: "connecting" }`
2. Creates a `ManagedRuntime` with WS protocol layer:
   ```typescript
   const layer = Layer.mergeAll(
     RpcClient.layerProtocolSocket(),
     RpcSerialization.layerNdjson,
     BrowserSocket.layerWebSocket(options.url),
   );
   const runtime = ManagedRuntime.make(layer);
   ```
3. Runs the streaming procedure via the runtime:
   ```typescript
   runtime.runPromise(
     Effect.scoped(
       Effect.gen(function* () {
         const client = yield* RpcClient.make(group);
         const stream = yield* (client as any)[options.procedure]({});
         yield* Stream.runForEach(stream, (value) =>
           Effect.sync(() => setState({ _tag: "connected", latest: value }))
         );
       })
     )
   );
   ```
4. On unmount: `runtime.dispose()` (closes WS connection)

The hook signature:
```typescript
function useRpcStream<Rpcs>(
  group: RpcGroup.RpcGroup<Rpcs>,
  options: { url: string; procedure: string }
): RpcStreamState<any, any>
```

Option A (chosen): `useRpcStream(group, { url, procedure: "WatchTodos" })` -- explicit procedure name, simpler for the common case (one stream per component).
