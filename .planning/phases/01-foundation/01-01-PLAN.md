In `packages/fresh/src/handlers.ts`:

1. Add the `EffectLike<A>` structural interface BEFORE the `HandlerFn` interface (around line 192).
   The interface uses the Effect v4 TypeId string as a computed property key:

   ```typescript
   /**
    * Structural type matching Effect v4 values. Used to extend
    * {@link HandlerFn} without importing `npm:effect` in Fresh core.
    *
    * This uses the Effect v4 TypeId string key for duck-type matching.
    * The type parameter `A` represents the expected success value type.
    */
   export interface EffectLike<A> {
     // deno-lint-ignore no-explicit-any
     readonly ["~effect/Effect"]: any;
   }
   ```

   Use `any` (not `unknown`) for the TypeId property value — Effect sets it to the
   effect's internal tag, not a useful type. The `deno-lint-ignore` comment suppresses
   the `no-explicit-any` rule.

2. Extend the `HandlerFn` interface return type union (currently at line 193-198).
   Add `EffectLike<Response | PageResponse<Data>>` as the fourth member:

   ```typescript
   export interface HandlerFn<Data, State> {
     (ctx: Context<State>):
       | Response
       | PageResponse<Data>
       | Promise<Response | PageResponse<Data>>
       | EffectLike<Response | PageResponse<Data>>;
   }
   ```

   Do NOT add an `E` type parameter to `HandlerFn` — this would cascade into
   `RouteHandler`, `HandlerByMethod`, `Route`, `Define`, breaking all existing routes.

In `packages/fresh/src/segments.ts`:

1. Add the module-level resolver variable and `setEffectResolver` function AFTER the
   existing imports (after line 13) and BEFORE the `RouteComponent` type (line 15):

   ```typescript
   // Effect resolver hook — null unless effectPlugin() registers one.
   // This keeps Fresh core free of any npm:effect dependency.
   let _effectResolver:
     | ((value: unknown, ctx: Context<unknown>) => Promise<unknown>)
     | null = null;

   /**
    * Register a resolver function that intercepts handler return values.
    * Called by effectPlugin() at setup time. The resolver receives the raw
    * handler return value and the request context; it should return the
    * value unchanged if it is not an Effect, or run the Effect and return
    * the unwrapped result.
    *
    * @internal Exported via `@fresh/core/internal` for plugin use only.
    */
   export function setEffectResolver(
     fn: (value: unknown, ctx: Context<unknown>) => Promise<unknown>,
   ): void {
     _effectResolver = fn;
   }
   ```

2. Modify the `renderRoute` function's tracer span callback (currently lines 166-190).
   Change the handler call at line 183 from:

   ```typescript
   return await fn(ctx);
   ```

   To:

   ```typescript
   let result: unknown = await fn(ctx);
   if (_effectResolver !== null) {
     result = await _effectResolver(result, ctx);
   }
   return result;
   ```

   The full tracer span callback should now read:

   ```typescript
   const res = await tracer.startActiveSpan("handler", {
     attributes: { "fresh.span_type": "fs_routes/handler" },
   }, async (span) => {
     try {
       let fn: HandlerFn<unknown, State> | null = null;
       if (isHandlerByMethod(handlers)) {
         if (handlers[method] !== undefined) {
           fn = handlers[method];
         } else if (method === "HEAD" && handlers.GET !== undefined) {
           fn = handlers.GET;
         }
       } else {
         fn = handlers;
       }

       if (fn === null) return await ctx.next();

       let result: unknown = await fn(ctx);
       if (_effectResolver !== null) {
         result = await _effectResolver(result, ctx);
       }
       return result;
     } catch (err) {
       recordSpanError(span, err);
       throw err;
     } finally {
       span.end();
     }
   });
   ```

   NOTE: The variable is now `res` (not `result`) at the outer scope — check that the
   existing `if (res instanceof Response)` check at line 192 still works. The `res`
   variable at line 166 receives the return from `tracer.startActiveSpan`, which is
   the inner `result`. This is correct — the type narrows from `unknown` via the
   existing `instanceof Response` and `typeof res.status` checks below.
