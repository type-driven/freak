# Plan: Per-app Effect runner + thread through all dispatch paths

## Goal
Replace the global `_effectResolver` singleton in `segments.ts` with a per-app `#effectRunner` field on `App`, and thread it through `renderRoute()`, `runMiddlewares()`, and `applyCommandsInner()` so that Effect handlers work via all registration paths (`app.route()`, `app.get()`, `app.post()`, `app.use()`).

## Context

@.planning/PROJECT.md
@.planning/ROADMAP.md
@.planning/phases/06-fresh-core-plumbing/06-RESEARCH.md

**Current state:**
- `_effectResolver` is a module-level global in `segments.ts` (line 17-19). It is set once by `effectPlugin()` via `setEffectResolver()`. This is last-writer-wins: two App instances sharing the same process clobber each other.
- `renderRoute()` in `segments.ts` (line 231) reads the global. Only `app.route()` goes through `renderRoute()` — `app.get()/post()` register raw middlewares that bypass it entirely.
- `runMiddlewares()` in `middlewares/mod.ts` has no Effect awareness. It expects middlewares to return `Response`. Effect-returning handlers/middlewares registered via `app.get()` or `app.use()` fail with "Expected a Response instance" error.

**Pattern to follow:** `app.ts` already uses the private field + static initializer pattern for `#onError` / `setErrorInterceptor` and `#getBuildCache` / `setBuildCache`. Replicate this exactly for `#effectRunner` / `setEffectRunner`.

**Key constraint:** `@fresh/core` must NOT import `npm:effect`. All Effect detection uses the `EffectLike` duck-type interface (string key `"~effect/Effect"` check). The `isEffectLike()` function must be defined in `handlers.ts` alongside the `EffectLike<A>` interface it checks for.

```typescript
/**
 * Duck-type check for Effect v4 values. Returns true if the value
 * has the `~effect/Effect` string key. No Effect import needed.

export function isEffectLike(value: unknown): value is EffectLike<unknown> {
  return (
    value !== null &&
    typeof value === "object" &&
    "~effect/Effect" in (value as object)
  );
}
```

Change to:

```typescript
const result = await next(ctx);
if (typeof result === "function") {  // lazy middleware
  middlewares[idx] = result;
  next = result;
  return await result(ctx);

// Resolve Effect-returning handlers/middlewares
if (effectRunner != null && isEffectLike(result)) {

  return resolved as Response;
```
