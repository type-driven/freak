    **services/TodoService.ts** -- Service definition + Layer:
    - Define `TodoService` using `ServiceMap.Service<Shape>("TodoService")`
    - Shape: `{ list, create, toggle, remove }` operations returning Effects
    - Error types: operations fail with `KvError`; toggle/remove also with `NotFoundError`
    - Export `TodoLayer` using `Layer.effect(TodoService, ...)`:
      - The effect opens KV via `Effect.tryPromise(() => Deno.openKv())`
      - Maps the opened kv handle to the service implementation object
      - `list()`: iterate `kv.list({ prefix: ["todos"] })`, return `Todo[]`
      - `create(text)`: generate UUID, set `["todos", id]`, return the new Todo
      - `toggle(id)`: get `["todos", id]`, flip `.done`, set, return updated Todo
      - `remove(id)`: delete `["todos", id]`, return void
      - Wrap all KV operations in `Effect.tryPromise` and `.pipe(Effect.mapError(...))`
        to produce `KvError` on failure
      - toggle/remove: check existence, return `NotFoundError` if not found
    - Export type: `export type TodoServiceR = ServiceMap.Service.Identifier<typeof TodoService>;`

    **services/layers.ts** -- Composed AppLayer:

    **main.ts** -- App assembly (NO mapError -- let failures propagate to Fresh _error.tsx):
    ```typescript
    import { App, staticFiles } from "@fresh/core";
    import { effectPlugin } from "@fresh/plugin-effect";
    import { AppLayer } from "./services/layers.ts";

    export const app = new App()
      .use(staticFiles())
      .use(effectPlugin({
        layer: AppLayer,
      }))
      .fsRoutes();
    ```
    IMPORTANT: Do NOT pass `mapError` to `effectPlugin()`. Without `mapError`, the
    resolver throws a standard Error wrapping the Effect Cause in `error.cause`. This
    thrown error propagates through Fresh's middleware chain to `_error.tsx`, where
    the error page can render a styled response and log `Cause.pretty()` server-side.
    Using `mapError` would short-circuit this by returning a JSON Response directly,
    bypassing Fresh's error page entirely.

    All imports must use the specifiers from deno.json imports -- e.g., `"effect"`,
    `"effect/Schema"`, `"@fresh/core"`, `"@fresh/plugin-effect"`.
