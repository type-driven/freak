**IMPORTANT:** Each test that creates an `EffectApp` will register signal listeners. Call `app.dispose()` in cleanup (or the `#cleanupSignals` returned from `registerSignalDisposal`) to remove them. If `dispose()` is exposed on `EffectApp`, call it in each test's teardown. Otherwise signal listener accumulation will cause issues.

Actually — looking at this more carefully, signal registration in tests is problematic. The `registerSignalDisposal` function calls `Deno.exit(0)` on signal receipt, which would kill the test process. For tests, we need to either:
- NOT register signal handlers (pass a flag to skip)
- OR register them but clean them up after each test

The `registerSignalDisposal` returns a cleanup function. The `EffectApp` stores it as `#cleanupSignals`. Ensure `EffectApp.dispose()` calls `this.#cleanupSignals()` FIRST (to remove listeners), then `await this.#runtime.dispose()`. Each test calls `await app.dispose()` at the end.

If `Deno.exit(0)` is still a concern (signal could fire during test), it's unlikely in a controlled test environment. The cleanup-before-dispose pattern is sufficient.

**packages/effect/tests/signal_server.ts:**

A minimal EffectApp server that prints "READY" to stdout when listening, then serves forever. Used by `signal_test.ts` as a subprocess target.

```typescript
import { createEffectApp } from "../src/mod.ts";
import { Layer } from "effect";

const app = createEffectApp({ layer: Layer.empty });
app.get("/health", () => new Response("ok"));

const handler = app.handler();

const server = Deno.serve(
  { port: 0, onListen: (addr) => console.log(`READY:${addr.port}`) },
  handler,
);

// Keep alive — the signal handler will call Deno.exit(0)
await server.finished;
```

**IMPORTANT:** Use `port: 0` to get a random free port. Print the port in the READY message so the test can parse it. The `onListen` callback fires once the server is actually bound.

**packages/effect/tests/signal_test.ts:**

SC-3 test: spawn signal_server.ts as a subprocess, wait for READY, send SIGTERM, assert exit code 0.

```typescript
import { assertEquals, assertMatch } from "jsr:@std/assert@1";

Deno.test("SC-3: SIGTERM causes clean shutdown with exit code 0", async () => {
  const cp = new Deno.Command(Deno.execPath(), {
    args: [
      "run",
      "--allow-net",
      "--allow-env",
      new URL("./signal_server.ts", import.meta.url).pathname,
    ],
    stdout: "piped",
    stderr: "piped",
  }).spawn();

  // Read stdout until we see "READY"
  const reader = cp.stdout.getReader();
  const decoder = new TextDecoder();
  let output = "";
  while (true) {
    const { value, done } = await reader.read();
    if (done) break;
    output += decoder.decode(value);
    if (output.includes("READY")) break;
  }
  reader.releaseLock();

  // Send SIGTERM
  cp.kill("SIGTERM");

  // Wait for process to exit
  const status = await cp.status;
  assertEquals(status.code, 0, `Expected exit code 0, got ${status.code}`);
});
```

Note: `Deno.ChildProcess.kill("SIGTERM")` sends SIGTERM to the child. The signal handler in `registerSignalDisposal` calls `runtime.dispose()` then `Deno.exit(0)`.

**Timeout:** Add a timeout to prevent hanging. Wrap the test body in a `Promise.race` with a 10-second timeout, or use Deno test's built-in `sanitizeOps: false` and `sanitizeResources: false` if needed. Actually, Deno tests have a default timeout. The subprocess test should complete in under 5 seconds. If it hangs, the test will fail with a timeout. Good enough.

Current code:
```typescript
import { App, HttpError, staticFiles } from "@fresh/core";
import { effectPlugin } from "@fresh/plugin-effect";
...

  .use(staticFiles())
  .use(effectPlugin({ layer: AppLayer, mapError: ... }))
```

New code:
```typescript
import { HttpError, staticFiles } from "@fresh/core";
import { createEffectApp } from "@fresh/effect";
import { Cause } from "effect";
import { AppLayer } from "./services/layers.ts";
import { NotFoundError } from "./services/errors.ts";

export const app = createEffectApp({
  layer: AppLayer,
  mapError: (cause) => {
    const defect = Cause.squash(cause as never);
    if (defect instanceof NotFoundError) {
      throw new HttpError(404);
    }
    throw new HttpError(500);
  },
})
  .use(staticFiles())
  .fsRoutes();
```

Note: `App` is no longer imported directly — `createEffectApp` creates it internally. `staticFiles()` from `@fresh/core` still works because `EffectApp.use()` accepts regular middlewares too.
